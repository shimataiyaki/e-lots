# E-LOTs

## 1. 概要
- **名称**：E-LOTs（公開名称：デジタルおみくじ）
- **リリース時期**：2026年4月（文化祭展示向け）
- **公開URL**：[https://shimataiyaki.github.io/e-mikuji/](https://shimataiyaki.github.io/e-lots/)
- **リポジトリ**：[https://github.com/shimataiyaki/e-mikuji](https://github.com/shimataiyaki/e-lots)

## 2. 開発の背景

### 2.1 着想と位置づけ
- 本作品は、**「部活動展示にも流用可能な個人制作プロダクト」** として開発された。
- 情報系部活の文化祭展示を想定しつつ、まずは個人のポートフォリオとして完成度を高めることを優先した。
- 静的サイト完結・紙カード連動という設計により、当日のネットワーク負荷や機材トラブルを回避できる見込みである。

### 2.2 世代別変遷（Gen.）

| 世代 | 主な変更点 |
| :--- | :--- |
| **Gen.1** | 最小構成。二進数（0000～1111）による抽選ロジックの実証。 |
| **Gen.2** | UIオマージュ版。恋みくじ v3 を参考に、朱色の札・縦書き・ホバーアニメーションを実装。 |
| **Gen.3** | 固定ヘッダー、ハンバーガーメニュー、16枚札、リセットボタン（後に廃止）を追加。 |
| **Gen.4（現行）** | **ポップアップ統合・数字漢字化・自動消滅廃止・左ボーダー削除・e-maとのデザイン統一。** |

## 3. 使用技術

## 5. 技術解説

### 5.1 抽選ロジック（Gen.4）
- 漢数字による抽選データ（`一`～`十八`、`四`と`十四`を文化的配慮から除外）を等確率で選択。
- 抽選中は札を無効化（`pointer-events: none`）し、連打を防止。
- 2.5秒の待機演出（`setTimeout`）後に、ポップアップを表示。

### 5.2 UI/UX（恋みくじオマージュ）
- **札のデザイン**：CSSのみで縦書き（`writing-mode: vertical-rl`）とホバー時の浮遊感を実現。角丸を排除した長方形。
- **ポップアップ統合**：抽選結果は画面下部ではなく、モーダルウィンドウ内に縦書きで表示。×ボタン・背景クリック・ESCキーで手動閉鎖（自動消滅なし）。
- **左ボーダー削除**：`e-ma`とのデザイン統一のため、案内文の左ボーダーを廃止。よりシンプルな表現に。
- **遊び方セクション**：「おみくじを引く」直下に「札を選んでタップしてください」を配置し、視線誘導を改善。

### 5.3 運用設計
- **Webと物理の融合**：画面上には漢数字のみを表示。運勢（大吉・吉など）は表示せず、部員がコード表と照合して紙カードを手渡す。これにより必ず会話が生まれる。
- **リセット動作**：ポップアップを閉じると同時に札の無効化が解除され、即座に次の抽選が可能。

## 6. 開発中の課題と解決策

| 課題 | 解決策 |
| :--- | :--- |
| 共用端末での連続利用時に前の結果が残る | ポップアップの×ボタンで状態を完全リセットする設計に変更 |
| 数字の「四」「十四」が不吉とされる懸念 | 抽選リストから除外し、代わりに「十七」「十八」を追加して16種類を維持 |
| `e-ma`とのデザイン言語の不一致 | 左ボーダーを削除し、両プロダクト間で完全な統一感を確保 |

## 7. 今後の展望
- 実際の文化祭展示を通じて、運営面の課題（紙カード在庫、待機時間の適正値など）を洗い出し、改善を図る。
- 効果音（鈴の音）の再生機能、結果の画像保存機能（`html2canvas`）の追加を検討中。

## 8. 謝辞／リスペクト
- 本作品のUIは「[恋みくじ v3](https://koi-mikuji.omikuji-do.com/v3/)」へのリスペクトに基づき、独自に実装したものです。
- プロジェクト名「e-mikuji」は、`e-ma`とのシリーズ統一、および「良い（e）みくじ」の意を込めたコードネームです。
